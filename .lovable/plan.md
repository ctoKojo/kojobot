# خطة إصلاح مشاكل السيشنات والسيشنات التعويضية

هذه خطة شاملة لإصلاح 5 مشاكل في نظام السيشنات. سيتم تنفيذ كل مشكلة بشكل منفصل.

---

## المشكلة 1: السيشن بتتحدد "مكتملة" اوتوماتيك لما المدرب يفتح الصفحة

### الوضع الحالي

- `SessionDetails.tsx` فيه `checkAndUpdateSessionStatus` بيشتغل لما الصفحة تتفتح وكل دقيقة
- بمجرد ما وقت السيشن ينتهي، بيحدث الحالة لـ "completed" اوتوماتيك بدون التأكد من إن المدرب عمل الأكشنز المطلوبة
- كمان `auto-complete-sessions` Edge Function بتعمل نفس الحاجة كل 15 دقيقة

### الحل المطلوب

1. **تغيير منطق الاكتمال**: السيشن تبقى "completed" فقط لما المدرب يكمل الـ 3 أكشنز (حضور + كويز + واجب) خلال فترة السيشن + ساعة سماح
2. **عداد فترة السماح**: بعد ما عداد السيشن يخلص، يظهر عداد تنازلي لمدة ساعة (Grace Period) للمدرب
3. **انذار بعد فترة السماح**: لو عدت الساعة بدون اكمال الأكشنز، يتحسب انذار ويتسجل غياب المدرب
4. **الادمن يقدر يعمل الأكشنز في أي وقت** بدون قيود زمنية

### التغييرات التقنية

#### قاعدة البيانات (Migration)

- اضافة عمود `attendance_mode` و `session_link` على جدول `sessions` (للمشكلة 5)

#### `SessionDetails.tsx`

- ازالة `checkAndUpdateSessionStatus` اللي بيعمل auto-complete
- اضافة حساب "فترة السماح" (session end + 60 دقيقة)
- اضافة عداد تنازلي يظهر للمدرب لما السيشن تخلص يوريه الوقت المتبقي من فترة السماح
- تعطيل ازرار الأكشنز للمدرب لو فترة السماح انتهت (الادمن يفضل شغال)
- لما المدرب يكمل آخر اكشن (حضور + كويز + واجب) خلال الوقت، السيشن تتحدد "completed" اوتوماتيك وحضور المدرب يتسجل

#### `auto-complete-sessions` Edge Function

- تعديل المنطق: بدل ما تحول السيشن لـ completed مباشرة بعد وقت النهاية، تنتظر ساعة بعد النهاية (grace period)
- بعد الساعة: لو الأكشنز مش مكتملة، تسجل انذار للمدرب + تسجل غيابه + تحدد السيشن completed

#### `Sessions.tsx`

- نفس التعديل على `handleMarkComplete` - يتحقق من اكتمال الأكشنز قبل السماح بالاكتمال اليدوي

## المشكلة 2: الغاء السيشن الكاملة مع السيشن التعويضية

### الوضع الحالي

- لما تلغي سيشن، بيسأل لو عايز سيشنات تعويضية
- لكن السيشن التعويضية مبتظهرش في قائمة السيشنات بشكل واضح
- السيشن الملغية ممكن يتعمل عليها أكشنز

### الحل المطلوب

1. لما يتم الغاء سيشن كاملة مع اختيار "تعويضية"، السيشن التعويضية تتحط جوا الجلسات اوتوماتيك
2. السيشن الملغية تظهر انها ملغاة ومقدرش اعمل عليها اي اكشنز

### التغييرات التقنية

#### `Sessions.tsx`

- تعطيل ازرار الأكشنز (Edit, Mark Complete, Delete) للسيشنات الملغية
- ابقاء زر "View Details" فقط

#### `SessionDetails.tsx`

- اخفاء "Quick Actions" للسيشنات الملغية
- اظهار بانر واضح ان السيشن ملغية ومفيش أكشنز متاحة

## المشكلة 4: تعديل معاد سيشن معينة مع ارسال نوتيفيكيشن

### الوضع الحالي

- `EditSessionDialog` موجود في `GroupDetails` ويسمح بتعديل التاريخ والوقت والحالة مع ارسال نوتيفيكيشن
- لكنه مش متاح في صفحة `Sessions.tsx` - هناك بس يقدر يعدل Topic/Status/Notes
- `RescheduleDialog` بيعدل كل السيشنات مش واحدة بعينها

### الحل المطلوب

- في `Sessions.tsx`، لما يضغط Edit على سيشن scheduled، يقدر يعدل التاريخ والوقت
- لما يتعدل معاد السيشن، يتبعت نوتيفيكيشن للطلاب

### التغييرات التقنية

#### `Sessions.tsx`

- اضافة حقول date و time في Edit Dialog
- اضافة منطق ارسال نوتيفيكيشن لطلاب المجموعة عند تغيير التاريخ/الوقت (نفس المنطق الموجود في `EditSessionDialog`)

## المشكلة 5: تحويل سيشن معينة لاونلاين مع لينك مخصوص

### الوضع الحالي

- `attendance_mode` و `session_link` موجودين على مستوى المجموعة فقط (`groups` table)
- مفيش طريقة لتحويل سيشن واحدة بعينها لاونلاين

### الحل المطلوب

1. اضافة `attendance_mode` و `session_link` على جدول `sessions`
2. لما يحول سيشن لاونلاين، يطلب اللينك
3. لما تكون سيشن تعويضية بتتكريت، يقدر يحدد اونلاين/اوفلاين
4. اللينك يظهر للطالب يوم السيشن
5. يتبعت نوتيفيكيشن للطلاب
6. الطالب وهو بياكد على معاد المجموعه او لا ياكد على انه عايز السيشن اونلاين ولا اوفلاين

### التغييرات التقنية

#### قاعدة البيانات (Migration)

```text
ALTER TABLE sessions ADD COLUMN attendance_mode text DEFAULT NULL;
ALTER TABLE sessions ADD COLUMN session_link text DEFAULT NULL;
```

- NULL = وراثة من المجموعة (السلوك الحالي)
- لما يكون محدد، السيشن ليها mode خاص

#### `Sessions.tsx` و `SessionDetails.tsx`

- اضافة زر "تحويل لاونلاين" في dropdown الأكشنز
- لما يضغط عليه يظهر dialog يطلب اللينك
- ارسال نوتيفيكيشن للطلاب
- عرض اللينك في SessionDetails يوم السيشن (اولوية session_link على group session_link)

#### `MakeupSessions.tsx`

- اضافة اختيار online/offline عند جدولة سيشن تعويضية
- لو online، يطلب اللينك

#### منطق عرض اللينك

- في `SessionDetails.tsx` و `StudentDashboard.tsx`: لو السيشن عندها `attendance_mode` و `session_link` خاص بيها، يتعرض بدل لينك المجموعة

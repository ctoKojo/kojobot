

# تحسين واجهة تقييم الطلاب (SessionEvaluationGrid)

## المشاكل الحالية
- الجدول مزدحم بأزرار صغيرة جدا (10px) يصعب الضغط عليها
- كل خلية تعرض كل مستويات الروبريك كأزرار متجاورة مما يسبب فوضى بصرية
- لا يوجد تمييز لوني واضح بين الصفوف (zebra striping)
- قسم الملاحظات والتاجز منفصل أسفل الجدول بعيدا عن سياق الطالب
- النسبة المئوية والمجموع يصعب تمييزهم بصريا
- لا يوجد progress indicator لكل طالب يوضح كم معيار تم تقييمه

## التحسينات المقترحة

### 1. تحويل من Table Layout الى Card-per-Student Layout
بدلا من جدول عريض يحتاج scroll أفقي، نعرض كل طالب في Card مستقل يحتوي على:
- اسم الطالب + progress bar يوضح نسبة اكتمال التقييم
- شبكة المعايير في grid واضح (2-3 أعمدة) بدل صف أفقي طويل
- الملاحظات والتاجز مباشرة تحت شبكة المعايير (بدل قسم منفصل)
- شريط الملخص (السلوك + كويز + واجب + المجموع + النسبة) في أسفل الكارد

### 2. تحسين أزرار الروبريك
- زيادة حجم الأزرار (من 10px الى 12-13px مع padding أكبر)
- ترتيبها أفقيا مع labels واضحة (مثلا "ضعيف | مقبول | جيد | ممتاز") بدل أرقام فقط
- تلوين تدريجي (gradient): أحمر للدرجة الأقل، أخضر للأعلى عند الاختيار
- إضافة hover state يعرض وصف المستوى

### 3. تحسينات بصرية
- Progress bar لكل طالب يوضح عدد المعايير المقيمة من الاجمالي
- ألوان مختلفة لمستوى الأداء في شريط الملخص (أخضر/أصفر/أحمر)
- أيقونة حالة الحفظ أوضح مع نص (محفوظ / جاري الحفظ)
- Zebra striping أو فواصل واضحة بين المعايير

### 4. تبديل العرض (Toggle View)
- إضافة زر تبديل بين "عرض الجدول" (الحالي المحسن) و "عرض البطاقات" (الجديد)
- العرض الافتراضي يكون البطاقات للموبايل والجدول للديسكتوب

### التغييرات الرئيسية في الكود

1. **إضافة state للـ view mode**: `useState<'cards' | 'table'>` مع `useIsMobile()` لتحديد الافتراضي

2. **مكون `StudentEvalCard`**: مكون جديد داخل نفس الملف يعرض بطاقة الطالب الواحد مع:
   - Header: اسم + progress bar + حالة الحفظ
   - Body: grid من المعايير (كل معيار في صف مع label + أزرار الروبريك + الدرجة)
   - أزرار الروبريك بحجم أكبر مع تلوين تدريجي (destructive -> warning -> secondary -> primary)
   - الملاحظات والتاجز مدمجة في البطاقة (نقلها من القسم المنفصل)
   - Footer: شريط ملخص الدرجات

3. **تحسين عرض الجدول**: إضافة zebra striping + أزرار أكبر + ألوان تدريجية

4. **Progress component**: استخدام `Progress` من shadcn لعرض نسبة اكتمال التقييم لكل طالب

### لا يوجد تغييرات في قاعدة البيانات
كل التحسينات في الـ frontend فقط - نفس الـ logic والـ API calls بدون أي تعديل.

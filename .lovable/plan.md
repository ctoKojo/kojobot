
# خطة إعادة هيكلة نظام الكويزات للإنستراكتور

## الفهم الصحيح للنظام

الإنستراكتور **لا يُنشئ كويزات** - بل يأخذها جاهزة من بنك الأسئلة الذي يديره الأدمن فقط.

## التغييرات المطلوبة

| الصفحة | الأدمن | الإنستراكتور | الطالب |

| تقارير الكويزات `/quiz-reports` | ✅ | ❌ | ❌ |

### 2. إنشاء صفحة جديدة للإنستراكتور: `MyInstructorQuizzes.tsx`

صفحة شاملة تتيح للإنستراكتور:

**أ. عرض الكويزات المتاحة من البنك**
- قائمة بجميع الكويزات النشطة في النظام
- معلومات كل كويز: الاسم، المدة، درجة النجاح، المستوى

**ب. إسناد كويز لمجموعة**
- اختيار كويز من القائمة
- اختيار مجموعة من مجموعاته فقط (filtered by `instructor_id = auth.uid()`)
- تحديد وقت البداية والنهاية التلقائية

**ج. إدارة الكويزات المُسندة**
- عرض الكويزات التي أسندها هو (filtered by `assigned_by = auth.uid()`)
- لكل إسناد: اسم المجموعة، عدد الطلاب، نسبة الإكمال

**د. رؤية النتائج التفصيلية**
- قائمة الطلاب: من سلم ومن لسه
- درجة كل طالب ونسبته
- بريفيو لإجابات الطالب مقارنة بالإجابة الصحيحة

### 5. استعلامات قاعدة البيانات

**جلب الكويزات المتاحة (للإسناد):**
```sql
SELECT * FROM quizzes 
WHERE is_active = true
ORDER BY created_at DESC
```

**جلب مجموعات الإنستراكتور (للإسناد):**
```sql
SELECT * FROM groups 
WHERE instructor_id = auth.uid() 
AND is_active = true
```

**جلب إسنادات الإنستراكتور مع النتائج:**
```sql
SELECT qa.*, q.*, g.*
FROM quiz_assignments qa
JOIN quizzes q ON qa.quiz_id = q.id
JOIN groups g ON qa.group_id = g.id
WHERE qa.assigned_by = auth.uid()
```

### 7. خطوات التنفيذ

1. **تعديل Sidebar**: جعل بنك الأسئلة للأدمن فقط
2. **إنشاء الصفحة الجديدة**: `MyInstructorQuizzes.tsx` بالتبويبين
3. **نقل منطق النتائج**: من `InstructorQuizResults.tsx` للصفحة الجديدة
4. **إضافة Route**: `/my-instructor-quizzes` للإنستراكتور
5. **تحديث Dashboard**: الروابط السريعة للإنستراكتور

### 8. الفوائد

- **تجربة موحدة**: صفحة واحدة تجمع كل ما يحتاجه الإنستراكتور
- **فصل الصلاحيات**: الأدمن يدير البنك، الإنستراكتور يستخدمه فقط
- **أمان أفضل**: الإنستراكتور لا يرى أدوات إنشاء/تعديل الكويزات

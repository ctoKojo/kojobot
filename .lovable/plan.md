# تطوير نظام الرسائل الداخلية ليكون اذكى

## المشاكل الحالية في النظام

- لا يوجد تحديث لحظي (Realtime) - يعتمد على polling كل 3-10 ثوان
- لا يوجد مؤشر كتابة (Typing Indicator)
- لا يوجد مؤشر قراءة الرسالة (Read Receipts / Seen)
- لا يمكن حذف الرسائل
- لا يوجد فصل تاريخي بين الرسائل (فواصل التاريخ)
- لا يوجد حالة اتصال (Online/Offline)
- لا يوجد إشعار صوتي عند وصول رسالة جديدة
- نتائج البحث في المحادثات لا تعرض دور المستخدم (طالب/مدرب/ادمن)

---

## التحسينات المقترحة

### 1. تحديث لحظي باستخدام Realtime (بدل Polling)

- استبدال `refetchInterval` بـ Supabase Realtime subscription
- الرسائل تظهر فورا عند الإرسال بدون تأخير
- تحديث قائمة المحادثات تلقائيا عند وصول رسالة جديدة

### 2. مؤشر الكتابة (Typing Indicator)

- إضافة جدول `typing_indicators` في قاعدة البيانات
- عرض "... يكتب" عندما الطرف الآخر يكتب رسالة
- المؤشر يختفي تلقائيا بعد 3 ثواني من آخر ضغطة

### 3. مؤشر القراءة (Read Receipts)

- عرض علامة "تم القراءة" (علامتين زرقاوين) اسفل الرسالة عندما يقرأها الطرف الآخر
- استخدام `last_read_at` الموجود فعلا لتحديد حالة القراءة

### 4. فواصل التاريخ بين الرسائل

- إضافة فاصل بتاريخ اليوم (اليوم، أمس، التاريخ الكامل) بين مجموعات الرسائل

### 5. عرض دور المستخدم في البحث والمحادثات

- إظهار badge صغير (طالب/مدرب/ادمن/استقبال) بجانب اسم المستخدم
- يسهل التعرف على الشخص خصوصا لو الاسماء متشابهة

### 6. إشعار صوتي + Push عند وصول رسالة

- تشغيل صوت تنبيه خفيف عند وصول رسالة جديدة والمستخدم في صفحة اخرى
- إرسال notification داخلي عبر جدول `notifications` الموجود

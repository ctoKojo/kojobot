
# نظام السيشنات التعويضية (Makeup Sessions)

## السيناريوهات المطلوبة

1. **سيشن مجموعة اتلغت** - كل الطلاب محتاجين تعويض
2. **طالب غاب** - محتاج سيشن تعويضية قبل السيشن الجاي
3. **لكل طالب 2 سيشن تعويضية فري لكل ليفل**

---

## التصميم

### 1. جدول جديد: `makeup_sessions`

| Column | Type | وصف |
|--------|------|-----|
| id | uuid | المعرف |
| student_id | uuid | الطالب |
| original_session_id | uuid (nullable) | السيشن الأصلية (اللي اتلغت أو غاب فيها) |
| group_id | uuid | المجموعة الأصلية |
| level_id | uuid (nullable) | الليفل (لحساب الحصة المجانية) |
| reason | text | السبب: `group_cancelled` / `student_absent` |
| status | text | الحالة: `pending` / `scheduled` / `completed` / `expired` |
| scheduled_date | date (nullable) | تاريخ السيشن التعويضية |
| scheduled_time | time (nullable) | وقت السيشن التعويضية |
| notes | text (nullable) | ملاحظات |
| is_free | boolean | هل مجانية ولا مدفوعة |
| created_at | timestamptz | تاريخ الإنشاء |
| completed_at | timestamptz (nullable) | تاريخ الإكمال |

### 2. التغييرات في الكود

#### أ. عند إلغاء سيشن (Sessions.tsx / SessionDetails.tsx)
- لما المدير أو المدرب يغير حالة سيشن لـ `cancelled`:
  - يظهر dialog يسأل: "هل تريد إنشاء سيشنات تعويضية لطلاب هذه المجموعة؟"
  - لو أيوا: يتم إنشاء سجل `makeup_session` لكل طالب في المجموعة بسبب `group_cancelled`
  - النظام يحسب تلقائي لو الطالب لسه عنده حصة مجانية ولا لأ

#### ب. عند تسجيل الغياب (Attendance.tsx)
- لما يتسجل طالب `absent`:
  - يظهر خيار "إنشاء سيشن تعويضية" جنب اسم الطالب
  - أو يتم إنشاءها تلقائي وتظهر في صفحة إدارة التعويضات

#### ج. صفحة جديدة: إدارة السيشنات التعويضية (`/makeup-sessions`)
- عرض كل السيشنات التعويضية المعلقة
- فلتر حسب: المجموعة، الطالب، الحالة، السبب
- للمدير: جدولة السيشن التعويضية (تحديد تاريخ ووقت)
- للمدير: تحديث الحالة (مكتملة / منتهية)
- عرض رصيد كل طالب من السيشنات المجانية (2 لكل ليفل)
- Badge تحذيري لما الطالب يستنفد الرصيد المجاني

#### د. إضافة في صفحة الطالب (StudentProfile.tsx)
- تاب جديد أو قسم يعرض:
  - عدد السيشنات التعويضية المجانية المتبقية
  - تاريخ السيشنات التعويضية (معلقة / مكتملة)

#### هـ. إضافة في الداشبورد (AdminDashboard.tsx)
- كارد يعرض عدد السيشنات التعويضية المعلقة

## منطق حساب الرصيد المجاني

```text
لكل طالب + ليفل:
  المجاني المستخدم = عدد السجلات في makeup_sessions
                     WHERE student_id = X
                     AND level_id = Y
                     AND is_free = true
  
  المتبقي = 2 - المجاني المستخدم
  
  لو المتبقي > 0 → السيشن الجديدة تكون is_free = true
  لو المتبقي <= 0 → السيشن الجديدة تكون is_free = false (مدفوعة)
```

## تدفق العمل

### سيناريو 1: إلغاء سيشن مجموعة
1. المدير يغير حالة السيشن لـ "ملغي"
2. يظهر dialog: "هل تريد إنشاء سيشنات تعويضية للطلاب؟"
3. لو وافق → النظام ينشئ سجل تعويض لكل طالب في المجموعة
4. كل سجل يتحسب تلقائي لو مجاني ولا لأ
5. تظهر التعويضات في صفحة إدارة التعويضات بحالة "معلق"

### سيناريو 2: غياب طالب
1. المدرب/المدير يسجل الحضور ويحط طالب "غائب"
2. بعد الحفظ يظهر زر "إنشاء تعويض" جنب الطلاب الغائبين
3. الضغط عليه ينشئ سجل تعويض للطالب
4. التعويض يظهر في صفحة الإدارة

### سيناريو 3: جدولة التعويض
1. المدير يفتح صفحة التعويضات
2. يختار سيشن معلقة ويحدد تاريخ ووقت
3. الحالة تتغير لـ "مجدولة"
4. بعد ما تتنفذ يحدثها لـ "مكتملة"

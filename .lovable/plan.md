
# خطة إضافة ميزة أونلاين/أوفلاين للطلاب والمجموعات

## نظرة عامة
إضافة خاصية تحديد نوع الحضور (أونلاين/أوفلاين) للطلاب والمجموعات مع:
- التحقق من التوافق عند إضافة طالب لمجموعة
- إضافة رابط الجلسة للمجموعات الأونلاين (يظهر للمدرب والطلاب)

## المتطلبات

### للطلاب
- إضافة حقل "نوع الحضور" (أونلاين/أوفلاين) عند إنشاء/تعديل الطالب
- الطالب الأونلاين لا يمكن إضافته لمجموعة أوفلاين والعكس

### للمجموعات
- إضافة حقل "نوع الحضور" (أونلاين/أوفلاين) عند إنشاء/تعديل المجموعة
- إضافة حقل "رابط الجلسة" (session_link) للمجموعات الأونلاين فقط
- الرابط يظهر للمدرب والطلاب في صفحة تفاصيل المجموعة

### التحقق من التوافق
- عند إضافة طالب لمجموعة، يجب أن يتطابق نوع الحضور
- تحديث منطق `getEligibleStudents` لتشمل هذا الشرط

## التفاصيل التقنية

### 1. تعديلات قاعدة البيانات

#### تعديل جدول `profiles` (الطلاب)
```text
+-------------------+---------+--------------------------------------+
| Column            | Type    | Description                          |
+-------------------+---------+--------------------------------------+
| attendance_mode   | text    | 'online' أو 'offline' (default: offline) |
+-------------------+---------+--------------------------------------+
```

#### تعديل جدول `groups`
```text
+-------------------+---------+--------------------------------------+
| Column            | Type    | Description                          |
+-------------------+---------+--------------------------------------+
| attendance_mode   | text    | 'online' أو 'offline' (default: offline) |
| session_link      | text    | رابط الجلسة (للمجموعات الأونلاين فقط) |
+-------------------+---------+--------------------------------------+
```

### 2. الملفات المطلوب تعديلها

#### صفحة الطلاب `src/pages/Students.tsx`
- إضافة حقل Select لاختيار نوع الحضور في فورم إضافة/تعديل الطالب
- عرض نوع الحضور في الجدول كـ Badge

#### Edge Function `supabase/functions/create-user/index.ts`
- إضافة حقل `attendance_mode` للـ request interface
- تخزين القيمة في جدول profiles

#### صفحة المجموعات `src/pages/Groups.tsx`
- إضافة حقل Select لاختيار نوع الحضور في فورم إضافة/تعديل المجموعة
- إضافة حقل Input لرابط الجلسة (يظهر فقط إذا كان النوع online)
- تحديث دالة `getEligibleStudents` لتشمل شرط تطابق نوع الحضور
- عرض نوع الحضور في الجدول

#### صفحة تفاصيل المجموعة `src/pages/GroupDetails.tsx`
- عرض رابط الجلسة بشكل واضح مع زر نسخ (للمجموعات الأونلاين)
- إظهار الرابط للمدرب والطلاب

#### ملف الترجمات `src/lib/i18n.ts`
- إضافة ترجمات للنصوص الجديدة

### 3. هيكل واجهة المستخدم

#### فورم إضافة طالب
```text
+-----------------------------------------+
| نوع الحضور *                            |
| [▼ اختر نوع الحضور                    ] |
|    • أونلاين                            |
|    • أوفلاين (حضوري)                    |
+-----------------------------------------+
```

#### فورم إضافة مجموعة
```text
+-----------------------------------------+
| نوع الحضور *                            |
| [▼ أونلاين                            ] |
+-----------------------------------------+
| رابط الجلسة (للمجموعات الأونلاين)       |
| [https://meet.google.com/abc-xyz      ] |
+-----------------------------------------+
```

#### عرض الرابط في تفاصيل المجموعة
```text
+---------------------------------------------------+
| 🔗 رابط الجلسة                                    |
| +-----------------------------------------------+ |
| | https://meet.google.com/abc-xyz   [📋 نسخ]   | |
| +-----------------------------------------------+ |
| [انضم للجلسة →]                                   |
+---------------------------------------------------+
```

### 4. منطق التحقق من التوافق

```text
الشروط الحالية لإضافة طالب لمجموعة:
1. ✓ تطابق نوع الاشتراك (subscription_type)
2. ✓ تطابق الفئة العمرية (age_group_id)
3. ✓ تطابق المستوى (level_id)
4. ✓ عدم وجود الطالب في مجموعة أخرى

الشرط الجديد:
5. ✓ تطابق نوع الحضور (attendance_mode)
```

## خطوات التنفيذ

1. **إنشاء Migration لقاعدة البيانات**
   - إضافة عمود `attendance_mode` لجدول `profiles`
   - إضافة عمودين `attendance_mode` و `session_link` لجدول `groups`

2. **تحديث Edge Function**
   - إضافة `attendance_mode` للـ CreateUserRequest interface
   - تخزين القيمة عند إنشاء طالب جديد

3. **تحديث صفحة الطلاب**
   - إضافة حقل اختيار نوع الحضور
   - عرض Badge لنوع الحضور

4. **تحديث صفحة المجموعات**
   - إضافة حقول نوع الحضور ورابط الجلسة
   - تحديث منطق التحقق من أهلية الطالب
   - عرض معلومات النوع والرابط

5. **تحديث صفحة تفاصيل المجموعة**
   - عرض رابط الجلسة مع أزرار النسخ والفتح

6. **إضافة الترجمات**
   - ترجمة جميع النصوص الجديدة

## ملاحظات إضافية

- **القيمة الافتراضية**: `offline` (أوفلاين) للحفاظ على التوافق مع البيانات الحالية
- **الرابط اختياري**: رابط الجلسة غير مطلوب حتى للمجموعات الأونلاين (يمكن إضافته لاحقاً)
- **أمان الرابط**: الرابط يظهر فقط للمدرب المسند للمجموعة والطلاب المسجلين فيها (عبر RLS الموجود)
